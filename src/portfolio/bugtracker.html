Deze BugTracker heb ik opgezet met Laravel.

Omdat een BugTracker vele kanten op kan, heb ik een project scope opgezet. Via Github is te zien hoe de scope in elkaar steekt. Schematisch ziet het er als volgt uit:

Een bug kan geregistreerd worden om aan te geven dat er iets aan de hand is. Hierbij heeft een bug de volgende kenmerken:

Categorie

Hiermee kan worden aangegeven in welke categorie de bug geplaatst kan worden.

  • Front-End
  • Back-End
  • Database
  • API
  • Website

Resolutie

Deze benaming kwam ik tegen in mijn onderzoek naar hoe bugtrackers zijn opgesteld. Hiermee kan worden aangegeven om wat voor soort bug het gaat.

  • Bug
  • New Feature
  • Improvement
  • Task

Status

Iedere bug heeft een status.

  • Open
  • Pending
  • In Progress
  • Resolved
  • Won´t fix
  • Closed
  • On Hold
  • ReOpen

Prioriteit

Niet geheel onbelangrijk, het aangeven van de prioriteit van een bug.

  • No Priority
  • Low
  • Normal
  • High
  • Immediate

Status transities

Bij iedere aanmelding van een bug, wordt de status "Open" meegegeven. Het is alleen door de administrator of employee mogelijk om de status te wijzigen.
